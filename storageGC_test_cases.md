# Storage GC 测试用例设计

## 1. 测试功能点概览

### 1.1 核心功能模块
- **触发策略**：文件无效率计算、阈值判定、文件合并策略
- **数据重写**：Base Bitmap过滤、RowId Mapping生成、新文件写入
- **Visibility同步**：Deletion Chain导出、RowId转换、头部插入
- **原子切换**：Metadata原子交换、双写映射注册
- **双写策略**：forwardMap/backwardMap、并发删除处理

### 1.2 关键算法验证
- Base Bitmap vs Deletion Chain的正确处理
- Padding机制的正确性
- 无效率的精确加权计算
- 文件名解析和Retina Node ID识别

## 3. 并发测试场景

### 3.1 读写并发场景
- **场景1**：GC过程中持续有查询请求访问旧文件
- **场景2**：GC过程中持续有新数据写入系统

### 3.2 多GC实例场景
- **场景4**：多个Retina节点同时执行GC

## 4. 测试环境配置

### 4.1 单元测试配置
```properties
# 测试专用配置
storage.gc.threshold=0.3  # 降低阈值便于测试
storage.gc.target.file.size=10485760  # 10MB测试文件
storage.gc.max.files.per.run=5
```

### 4.2 集成测试配置
```properties
# 生产环境配置
storage.gc.threshold=0.5
storage.gc.target.file.size=134217728  # 128MB
storage.gc.interval=3600000
storage.gc.max.files.per.run=10
```

## 5. 测试数据准备

### 5.1 测试文件生成策略
- **正常文件**：包含有效数据和部分删除记录
- **边界文件**：无效率刚好在阈值附近的文件
- **异常文件**：损坏文件、权限异常文件
- **大文件**：超过配置目标大小的文件

### 5.2 测试数据验证方法
- **文件一致性**：比较新旧文件的数据内容
- **Visibility正确性**：验证删除记录的完整迁移
- **性能指标**：监控GC执行时间和资源使用
